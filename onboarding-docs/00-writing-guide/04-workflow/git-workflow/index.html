ワークフローとリファレンス

Git操作

コミット、プッシュ、Pull Requestの作成方法
最終更新: 2026/2/9

ドキュメントの変更を保存し、チームと共有するためのGit操作を解説します。

コミット

作業の区切りごとに、変更をコミット（保存）します。

VS Codeでのコミット

ソース管理タブを開く

VS Code左側のアイコンから「ソース管理」タブを開きます。

変更をステージング

変更したファイルの横にある+ボタンをクリックして、ステージングエリアに追加します。

すべてのファイルをステージングする場合は、「変更」の横にある+ボタンをクリックします。

コミットメッセージを入力

上部のテキストボックスに、コミットメッセージを入力します。

良いコミットメッセージの例：

watsonx.aiのインスタンス作成手順を追加

悪いコミットメッセージの例：

更新
何を変更したかが分かるメッセージを書きましょう。

コミット

「コミット」ボタンをクリックします。

コマンドラインでのコミット

変更をステージング：

git add .

コミット：

git commit -m "コミットメッセージ"

プッシュ

ローカルのコミットをリモートリポジトリに送信します。

VS Codeでのプッシュ

ソース管理タブの「...」メニューから「プッシュ」を選択します。

または、画面下部の「同期」ボタンをクリックします。

コマンドラインでのプッシュ

git push
初回プッシュ時は、以下のコマンドが必要な場合があります：
git push -u origin ブランチ名

直前のコミットの取り消し

間違えてコミットした場合、直前のコミットを取り消せます。

コマンドライン

git reset --soft HEAD^
このコマンドは、コミットを取り消しますが、変更内容は保持されます。

Pull Requestの作成

作成したドキュメントをdevelopブランチに統合するため、Pull Requestを作成します。

事前準備：developブランチの最新化

統合前に、元のdevelopブランチの差分を取り込みます。

developブランチに切り替え：

git checkout develop

最新の変更を取得：

git pull

作業ブランチに戻る：

git checkout -

developブランチの変更をマージ：

git merge develop
git checkout -は、直前のブランチに戻るコマンドです。

Pull Requestの作成手順

Pull Requestページにアクセス

Pull Requestページを開きます。

New pull requestをクリック

右上の「New pull request」ボタンをクリックします。

ブランチを選択

  • base: develop
  • compare: doc/your-branch（あなたの作業ブランチ）

正しく選択されていることを確認します。

Create pull requestをクリック

「Create pull request」ボタンをクリックします。

タイトルを入力

タイトル例：

watsonx.aiのオンボーディングガイドを追加

レビュアーを追加（オプション）

必要に応じて、レビュアーを追加します。

Create pull requestをクリック

「Create pull request」ボタンをクリックして、Pull Requestを作成します。

Pull Requestのマージ

レビューが完了し、問題がなければ、Pull Requestをマージします。

Merge pull requestをクリック

Pull Requestページの「Merge pull request」ボタンをクリックします。

Confirm mergeをクリック

「Confirm merge」ボタンをクリックして、マージを確定します。

ブランチの削除（オプション）

マージ後、作業ブランチが不要な場合は「Delete branch」ボタンをクリックして削除します。

修正が必要な場合は、ブランチを残しておくこともできます。

コンフリクトの解決

developブランチと作業ブランチで同じファイルが変更されている場合、コンフリクトが発生します。

コンフリクトの確認

git merge develop

コンフリクトが発生すると、以下のようなメッセージが表示されます：

CONFLICT (content): Merge conflict in ファイル名
Automatic merge failed; fix conflicts and then commit the result.

コンフリクトの解決

コンフリクトファイルを開く

VS Codeでコンフリクトが発生したファイルを開きます。

変更を選択

VS Codeは、以下の選択肢を表示します：

  • Accept Current Change：作業ブランチの変更を採用
  • Accept Incoming Change：developブランチの変更を採用
  • Accept Both Changes：両方の変更を採用
  • Compare Changes：変更を比較

適切な選択肢をクリックします。

コミット

コンフリクトを解決したら、変更をコミットします。

git add .
git commit -m "コンフリクトを解決"

ベストプラクティス

1. 小まめなコミット

良い例：

git commit -m "IBM Cloudログイン手順を追加"
git commit -m "サブスクリプションコード手順を追加"
git commit -m "画像のパスを修正"

悪い例：

git commit -m "全部完成"

2. プッシュ前の確認

変更内容を確認：

git status

差分を確認：

git diff

コミット履歴を確認：

git log --oneline

3. Pull Request前の確認

  • 開発サーバーでプレビュー確認
  • 画像のパスが正しい（/製品名/image.png）
  • リンクが機能する
  • コミットメッセージが適切
  • developブランチの最新変更を取り込んだ
