デプロイ

ドキュメントを本番環境に公開する方法
最終更新: 2026/2/9

作成したドキュメントを本番環境に公開する方法を解説します。

デプロイ先

以下のリポジトリのGitHub Pagesで公開できます。

  1. IBM/japan-technology リポジトリ - 外部公開用
  2. customer-success-japan/onboarding-docs リポジトリ - 社内公開・動作検証用

IBM/japan-technology での公開

外部公開用のリポジトリです。

必要な権限

IBM/japan-technologyリポジトリへのpushには権限が必要です。 権限付与には研修+マネージャー承認が必要です。

事前準備

japan-technologyリポジトリのクローン

onboarding-docsと同じ階層にクローンします。

onboarding-docsの親ディレクトリに移動：

cd ..

japan-technologyをクローン：

git clone git@github.com:IBM/japan-technology.git

ディレクトリ構造の確認：

ls

以下のような構造になります：

onboarding-docs/
japan-technology/

ブランチの切り替え

japan-technologyリポジトリでgh-pagesブランチに切り替えます。

cd japan-technology
git checkout gh-pages

デプロイ手順

onboarding-docsで最新の変更を取得

cd ../onboarding-docs
git checkout develop
git pull

ビルドの実行

npm run build:jpn-tech

このコマンドは以下を実行します：

  1. NUXT_APP_BASE_URL=/japan-technology/onboarding-docs/でビルド
  2. .output/publicを../japan-technology/onboarding-docsにコピー
  3. 一時ファイルをクリーンアップ
ビルドには数分かかる場合があります。

japan-technologyでコミットとプッシュ

cd ../japan-technology
git add .
git commit -m "Update onboarding-docs"
git push

デプロイの確認

GitHub Actionsページでデプロイ状況を確認します。

Jobが完了すると、公開ページに反映されます。

デプロイには数分かかる場合があります。

customer-success-japan/onboarding-docs での公開

社内公開・動作検証用のリポジトリです。

事前準備

このリポジトリのmainブランチは保護されており、直接pushすることはできません。デプロイ作業を行うには、リポジトリ管理者からmainブランチへのpush権限を取得してください。

権限取得後、以下の手順でデプロイを実行します。

デプロイ手順

GitHubでdevelopブランチをmainにマージ

mainブランチに切り替え

git checkout main
git pull

ビルドの実行

npm run build:ghp-int

このコマンドは以下を実行します：

  1. NUXT_APP_BASE_URL=/customer-success-japan/onboarding-docs/でビルド
  2. .output/publicをdocsディレクトリにコピー
  3. 一時ファイルをクリーンアップ

コミットとプッシュ

git add docs
git commit -m "Update docs"
git push

デプロイの確認

GitHub Pagesの設定で、docsディレクトリが公開されます。

ビルドコマンド

開発用

開発サーバーの起動：

npm run dev

デプロイ用

IBM/japan-technology用：

npm run build:jpn-tech

customer-success-japan用：

npm run build:ghp-int

その他

PDF生成：

npm run build:pdf

最後に、クイックリファレンスで、全設定パラメータを確認しましょう。
