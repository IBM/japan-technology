トップページの作成

製品ガイドのトップページ（index.md）の作成
最終更新: 2026/2/9

製品のトップページ（index.md）は、製品ガイドの入り口となる重要なページです。

index.mdの役割

トップページは以下の役割を持ちます：

  • 製品の概要説明
  • オンボーディングプロセスの全体像
  • 各セクションへのナビゲーション（目次）

ファイルの作成

製品ディレクトリ直下にindex.mdを作成します。

watsonx.aiの場合：

touch content/watsonx-ai/index.md

Frontmatterの設定

ファイルの先頭に、YAMLフォーマットでメタデータを記述します。

基本的な設定

content/製品名/index.md
---
title: "製品名"
description: "製品の簡潔な説明"
---

パラメータ詳細

title
string required
ページのタイトルです。ページ上部とブラウザタブに表示されます。
title: "watsonx.ai"
.product.ymlのtitleと同じにするのが一般的ですが、異なる表現も可能です。
description
string required
ページの説明です。タイトル下のグレーテキストとして表示されます。
description: "Onboarding Guide"

設定例

content/watsonx-ai/index.md
---
title: "watsonx.ai"
description: "Onboarding Guide"
---

コンテンツの構成

推奨される構成

---
title: "製品名"
description: "Onboarding Guide"
---

3. オンボーディングプロセス

## Onboarding Process

製品名のオンボーディングプロセスは、以下の領域に分かれています：

:toc

それぞれのステップに関する情報、ガイド、参考リンクについては、該当するリンクを選択してください。

:tocコンポーネントで、自動的にセクションへのリンクを生成します。

:tocコンポーネント

:tocは、製品ガイドのセクション一覧を自動生成するカスタムコンポーネントです。

基本的な使い方

:toc

インライン記法で記述するだけで、ナビゲーション構造からセクションリンクを生成します。

表示される内容

:tocは以下の情報を表示します：

  • 製品ガイド内の全セクション（ディレクトリ）
  • 各セクション内のページ一覧（製品トップページ（index.md）を除く）
セクションとページを作成すると、自動的に:tocに反映されます。手動でリンクを追加する必要はありません。

プレビューでの確認

開発サーバーを起動して、以下を確認します：

npm run dev

チェックリスト

トップページの作成が完了したら、以下を確認してください：

  • index.mdを作成した
  • Frontmatterにtitleとdescriptionを設定した
  • 製品概要を記述した
  • :tocコンポーネントを配置した
  • 開発サーバーで以下の項目を確認した
    • タイトルと説明が正しく表示される
    • :tocが表示される位置が適切
    • リンクが正しく機能する
:tocは、セクションとページを作成した後に内容が表示されます。この段階では空の状態で表示されます。
