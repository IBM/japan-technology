製品の作成

新しい製品ガイドのディレクトリと設定ファイルの作成
最終更新: 2026/2/9

製品ガイドを作成する最初のステップは、製品のディレクトリと設定ファイルを作成することです。

ディレクトリの作成

content/ディレクトリ内に、製品名のディレクトリを作成します。

ディレクトリ名の規則

  • 小文字とハイフンを使用（例：watsonx-ai, instana）
  • スペースは使用しない
  • わかりやすい名前にする

watsonx.aiの例：

mkdir content/watsonx-ai

Instanaの例：

mkdir content/instana

Turbonomicの例：

mkdir content/turbonomic

.product.ymlの作成

製品ディレクトリ内に.product.ymlファイルを作成します。このファイルは、トップページの製品カードに表示される情報を定義します。

基本的な設定

content/製品名/.product.yml
title: "製品名"
description: "製品の簡潔な説明（1-2行）"
group: "グループ名"
image: "アイコンまたは画像パス"
order: 1

パラメータ詳細

title
string required
製品の表示名です。トップページのカードとナビゲーションに表示されます。
title: "IBM Instana Observability"
description
string required
製品の簡潔な説明です。トップページのカードに表示されます。
1-2行程度の簡潔な説明が推奨されます。詳細は製品のindex.mdに記載します。
description: "AIと自動化の力を活用して、アプリケーション・スタック全体の問題をプロアクティブに解決"
group
string required
製品が属するグループ名です。トップページでグループごとに製品が分類されます。
group: "watsonx"
利用可能なグループの例：
  • watsonx - watsonx製品群
  • IT Automation - Instana, Turbonomicなど
現在あるグループについては.groups.ymlをご覧ください。適切なものがなければ無ければ追加も可能です。
新しいグループを追加する場合は、.groups.ymlの編集を参照してください。
image
string required
製品カードに表示されるアイコンまたは画像です。

オプション1：Carbonアイコン（推奨）
image: "i-carbon-ibm-cloud"
IBM製品のドキュメントでは、Carbonアイコンの使用を推奨します。アイコンの検索方法はアイコンについてを参照してください。

オプション2：カスタム画像
image: "/product-icon/instana.png"
public/product-icon/ディレクトリに画像を配置し、パスを指定します。
推奨サイズ： 40x40px（正方形） 推奨フォーマット： PNG（透過背景）
order
number
グループ内での表示順序です。数字が小さいほど上に表示されます。
order: 1
省略すると辞書順（ディレクトリ名のアルファベット順）で表示されますが、既存の製品との兼ね合いを考慮してorderで適切な順番に並べることを推奨します。
forQuote
boolean
trueに設定すると、この製品は「引用専用」となり、トップページの製品一覧に表示されません。
forQuote: true
用途：
  • 共通手順（IBM Cloudインスタンス作成など）
  • 他の製品ガイドから:page-compで参照されるページ
forQuote: trueの製品は、開発環境でのみ一覧に表示され、QUOTEバッジが付きます。
dev
boolean
trueに設定すると、この製品は「開発中」となり、開発環境でのみ表示されます。
dev: true
用途：
  • 作成中の製品ガイド
  • レビュー中のコンテンツ
dev: trueの製品は、本番環境では一覧に表示されません。開発環境ではDEVバッジが付きます。

設定例

通常の製品
content/watsonx-ai/.product.yml
title: "watsonx.ai"
description: "AIをシンプルかつスケーラブルに構築して導入可能にする統合AI開発スタジオ"
group: "watsonx"
image: "/product-icon/watsonx-ai.png"
order: 1
引用専用の共通ページ
content/cloud-instance/.product.yml
title: "IBM Cloud 共通手順"
description: "IBM Cloud インスタンス作成の共通手順（引用専用）"
group: "共通"
image: "i-carbon-ibm-cloud"
forQuote: true
order: 999
開発中の製品
content/new-product/.product.yml
title: "新製品（開発中）"
description: "現在作成中の製品ガイドです"
group: "watsonx"
image: "i-carbon-cube"
dev: true
order: 99

.groups.ymlの編集

新しいグループを追加する場合は、content/.groups.ymlを編集します。

ファイルの場所

content/.groups.yml

グループの追加

content/.groups.yml
groups:
  - name: "ドキュメント"
    order: 0
  - name: "watsonx"
    order: 1
  - name: "新しいグループ"  # 追加
    order: 4              # 追加
  - name: "共通"
    order: 98
  - name: "開発用"
    order: 100
    dev: true

パラメータ

name（必須）

グループの表示名です。

name: "watsonx"
order（必須）

トップページでのグループの表示順序です。数字が小さいほど上に表示されます。

order: 1
dev（オプション）

trueに設定すると、このグループは開発環境でのみ表示されます。

dev: true

アイコンについて

アイコンはCarbonアイコンを使用してください。

Carbonアイコンの検索

  1. Icones.js.org - Carbonにアクセス
  2. 検索ボックスでアイコンを検索
  3. アイコンをクリックして名前をコピー
  4. i-プレフィックスを付けて使用
image: "i-carbon-ibm-cloud"
image: "i-carbon-watson-health"
image: "i-carbon-data-base"
image: "i-carbon-cube"

チェックリスト

製品の作成が完了したら、以下を確認してください：

  • 製品ディレクトリを作成した
  • .product.ymlを作成し、全ての必須パラメータを設定した
  • グループ名が正しい（必要に応じて.groups.ymlを編集した）
  • アイコンまたは画像が正しく表示される
  • orderで表示順序を設定した
  • 開発サーバーでトップページに製品カードが正しく表示された
