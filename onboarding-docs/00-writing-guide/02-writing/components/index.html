コンテンツ執筆

コンポーネント一覧

ドキュメントで使用できるコンポーネント
最終更新: 2026/2/9

Nuxt UIのコンポーネントを使用して、リッチなドキュメントを作成できます。

Callout（注意書き）

重要な情報を目立たせるボックスです。

基本的な使い方

::note
追加情報をお伝えします。
::

::tip
役立つ提案です。
::

::warning
この操作は予期しない結果を招く可能性があるため、注意してください。
::

::caution
この操作は元に戻せません。
::

プレビュー

追加情報をお伝えします。
役立つ提案です。
この操作は予期しない結果を招く可能性があるため、注意してください。
この操作は元に戻せません。

カスタマイズ

iconとcolorプロパティでカスタマイズできます。

::callout{icon="i-carbon-information" color="blue"}
カスタムアイコンと色を使用した注意書きです。
::

Badge（バッジ）

バージョン番号やステータスを表示します。

基本的な使い方

::badge
v3.0.0
::

プレビュー

v3.0.0

インライン使用

新機能 ::badge
NEW
:: が追加されました。

Card と CardGroup

コンテンツをカード形式で表示します。

基本的な使い方

::card
---
title: "カードのタイトル"
icon: "i-carbon-cube"
to: "/path/to/page"
---
カードの説明文です。
::

CardGroupでグリッド表示

::card-group
  :::card
  ---
  title: "ドキュメント"
  icon: "i-carbon-book"
  to: "/docs"
  ---
  ドキュメントを確認
  :::

  :::card
  ---
  title: "API"
  icon: "i-carbon-code"
  to: "/api"
  ---
  API リファレンス
  :::
::

プレビュー

ドキュメント

執筆ガイドを確認

Markdown

Markdown記法を学ぶ

Tabs（タブ）

複数のコンテンツをタブで切り替えます。

基本的な使い方

::tabs
  :::tabs-item{icon="i-carbon-earth-americas" label="ダラス"}
  `ダラス (us-south)` を選択する
  :::

  :::tabs-item{icon="i-carbon-earth-southeast-asia" label="東京"}
  `東京 (jp-tok)` を選択する
  :::
::

プレビュー

ダラス (us-south) を選択する

東京 (jp-tok) を選択する

Steps（ステップ）

手順を番号付きで表示します。

基本的な使い方

::steps{level="4"}
#### 環境構築

Node.jsをインストールします。

```bash
nvm install 22
```

#### 依存関係のインストール

プロジェクトの依存関係をインストールします。

```bash
npm install
```

#### 開発サーバーの起動

開発サーバーを起動します。

```bash
npm run dev
```
::

プレビュー

環境構築

Node.jsをインストールします。

nvm install 22

依存関係のインストール

プロジェクトの依存関係をインストールします。

npm install

開発サーバーの起動

開発サーバーを起動します。

npm run dev
levelプロパティで、ステップに使用する見出しレベルを指定します（デフォルト：3）。

Accordion（アコーディオン）

折りたたみ可能なコンテンツを作成します。

基本的な使い方

::accordion
  :::accordion-item
  ---
  icon: "i-carbon-help"
  label: "よくある質問1"
  ---
  回答内容をここに記述します。
  :::

  :::accordion-item
  ---
  icon: "i-carbon-help"
  label: "よくある質問2"
  ---
  回答内容をここに記述します。
  :::
::

プレビュー

Frontmatterが正しく記述されているか確認してください。---で囲まれ、titleが設定されている必要があります。

画像のパスが正しいか確認してください。/製品名/image.pngの形式で、/から始まる必要があります。

Collapsible（折りたたみ）

長いコンテンツを折りたたみます。

基本的な使い方

::collapsible
長いコンテンツをここに記述します。
デフォルトで折りたたまれた状態で表示されます。
::

プレビュー

パラメータデフォルト型説明
titlestringページのタイトル
descriptionstringページの説明
navigation.iconstringナビゲーションアイコン
navigation.titlestringナビゲーション表示名
ordernumber表示順序

Field と FieldGroup

APIパラメータやプロパティを説明します。

基本的な使い方

::field-group
  :::field{name="title" type="string"}
  ページのタイトルです。必須パラメータです。
  :::

  :::field{name="description" type="string"}
  ページの説明です。必須パラメータです。
  :::

  :::field{name="order" type="number"}
  デフォルトは`1` - 表示順序を指定します。
  :::
::

プレビュー

title
string
ページのタイトルです。必須パラメータです。
description
string
ページの説明です。必須パラメータです。
order
number
デフォルトは1 - 表示順序を指定します。

Icon（アイコン）

アイコンを表示します。

基本的な使い方

:icon{name="i-carbon-checkmark"}
:icon{name="i-carbon-close"}
:icon{name="i-carbon-ibm-cloud"}

プレビュー

アイコンの探し方

Icones.js.org - Carbonで検索し、i-プレフィックスを付けて使用します。

Kbd（キーボード）

キーボードショートカットを表示します。

基本的な使い方

:kbd{value="meta"} :kbd{value="K"}

プレビュー

K

コンポーネントのネスト

コンポーネントは入れ子にできます。コロンの数を増やして階層を表現します。

::tabs
  :::tabs-item{label="タブ1"}
    ::::note
    タブ内の注意書き
    ::::
  :::

  :::tabs-item{label="タブ2"}
    ::::tip
    タブ内のヒント
    ::::
  :::
::
ネストが深くなりすぎると可読性が下がります。3階層程度までに抑えることを推奨します。

次は、コードブロックで、コードの表示方法を詳しく学びます。
