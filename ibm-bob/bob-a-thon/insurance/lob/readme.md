# Bob-a-thon (AIワークショップ)
## 保険業界ビジネス担当者向け
* 最終更新日: 2026/3/13
* Author: Akira Onishi (AI Lab Japan Office, IBM Japan)

## 📋 ワークショップ概要

### 目的
- **AI駆動開発のパワーを体感**：IBM Bobを使った開発の速さと可能性を実感
- **ユーザー体験重視の開発**：コードの正確性よりもUXの素晴らしさを追求
- **ビジネス側とIT側の視点の確認**：異なる視点からのアプローチを比較し、相互理解を深める
- **実践的なプロトタイピング**：2時間で動くモックアップを作成

### 状況の想定
* 少人数で新規の保険会社「GeekLife保険(仮）」を起業します。一人ひとりがリーダーであり、実務家です。
* テクノロジーを活用した新会社という設定です。AI駆動開発を採用し、IBM Bobが皆さんのパートナーです。
* 企業名、企業のミッション(存在意義)、ビジョン(理想)、バリュー(行動指針)、企業ブランドの色・フォント、主要商品の名称などについては、参加者で相談して、IBM Bobと壁打ちしながら、決めてください。
* 営業、マーケティング、契約・サービス管理、人事、これら4つの部門に分かれて作業します。1部門あたりの人数が2名を超えないようにしてください。
* 業務に必要なアプリを内製で整えていきます。ただし、このワークショップではアプリの完成品を作る必要はありません。
* 業務上のユーザー体験をモックアップを通じて確認できるようにして、具体的な業務遂行に向けて準備するのが目的です。
* 困ったことがあれば、まずはIBM Bobに相談してください。
* 他のメンバーと相談したり、一緒に考えるのは構いません。ただし、成果物は、同じ役割の人と、IBM Bobと一緒に作り上げてください。
* IBM Bobの操作がわからない、モックアップの修正において困ったことがあれば、ワークショップのサポーターにお声がけください。

### The Geek Way (ギーク思考）について
* 参考書籍: [ギーク思考 圧倒的な結果を出す型破りな思考法](https://bookplus.nikkei.com/atcl/catalog/25/03/25/01931/)
- ギーク思考の４つの規範について、ワークショップを通じて体感してください。
* スピード: 綿密な計画よりも、結果を出すための素早い反復

  まずは試してみて、上手くいかなかったら、別のアプローチを試してみるようにします。上手くいったら、さらに改善を試みます。
  特にIBM Bob (AIエージェント)との会話を何度も往復することが重要です。
  
* オーナーシップ: 共通のゴールを目指して、自律的に活動

　状況設定にある通り、少人数で新規の保険会社を起業します。一人ひとりがリーダーであり、実務家です。
  業務のイメージを固めるためにモックアップを作りましょう。IBM Bobが皆さんのパートナーです。
 
* サイエンス: 実験からデータを集め、証拠を元に議論を実践

　2時間程度のワークショップなので、詳細なデータを集めるというよりは、試行した結果から、よりブラッシュアップすることを意識してください。

* オープンネス: 情報を徹底して共有し、議論や方向転換を歓迎
 
 アイディアを共有する、出来上がったモックアップを他の人に操作してもらう、など、周りからフィードバックをもらうことを心がけてください。
 またフィードバックを求められたら、積極的に参加して、より良くするための意見を伝えましょう。


## 🎯 各業務領域の課題設計

### 設計方針
1. **2時間で完成可能な範囲**：シンプルだが価値のある機能に絞る
2. **ユーザー体験重視**：見た目の美しさ、使いやすさを評価
3. **実務との関連性**：実際の業務課題を反映
4. **創造性の余地**：正解が一つではない、工夫の余地がある課題

### 1. 営業チーム：「顧客訪問支援システム」

#### 課題カード
```
【GeekLife保険 営業チームの課題】

新規保険会社として、営業担当者が効率的に顧客訪問を行い、
適切な保険商品を提案できるシステムが必要です。

■ ペルソナ
- 営業担当者：田中さん（35歳、営業経験5年）
- 1日に3-5件の顧客訪問をこなす
- 移動中や訪問前に顧客情報を確認したい

■ 解決したい課題
- 顧客の基本情報（年齢、家族構成、職業）を素早く確認
- 顧客に最適な保険商品を提案
- 訪問履歴と次回アクションを記録

■ 必須機能（最低限これだけは実現）
1. 顧客リスト表示（名前、年齢、家族構成）
2. 顧客詳細画面（基本情報 + 訪問履歴）
3. おすすめ保険商品の表示

■ あると嬉しい機能（時間があれば）
- 顧客の検索・フィルター機能
- 訪問予定のカレンダー表示
- 簡単な提案書生成

■ UX重視ポイント
- モバイルでも見やすいデザイン
- 情報が一目でわかるビジュアル
- 直感的な操作性
```

#### 期待される成果物
- **画面構成**：顧客リスト → 顧客詳細 → 商品提案
- **デモシナリオ例**：「田中さんが午後の訪問前に、山田様（45歳、妻・子供2人）の情報を確認し、家族向け医療保険を提案する」

### 2. マーケティングチーム：「キャンペーン管理ダッシュボード」

#### 課題カード
```
【GeekLife保険 マーケティングチームの課題】

新規顧客獲得のため、様々なマーケティングキャンペーンを実施します。
各キャンペーンの効果を可視化し、素早く意思決定できるシステムが必要です。

■ ペルソナ
- マーケティング担当者：佐藤さん（32歳、デジタルマーケティング経験3年）
- 複数のキャンペーンを同時に管理
- データに基づいた意思決定を重視

■ 解決したい課題
- 実施中のキャンペーン一覧を把握
- 各キャンペーンの成果（リード数、コンバージョン率）を確認
- 効果の高いキャンペーンを特定

■ 必須機能（最低限これだけは実現）
1. キャンペーン一覧表示（名前、期間、ステータス）
2. キャンペーン詳細画面（目標、実績、進捗率）
3. 簡単なグラフ・チャート表示

■ あると嬉しい機能（時間があれば）
- キャンペーン比較機能
- リアルタイム更新のダッシュボード
- アラート機能（目標未達時など）

■ UX重視ポイント
- データが直感的に理解できるビジュアライゼーション
- 重要な指標が一目でわかるデザイン
- カラフルで見やすいUI
```

#### 期待される成果物
- **画面構成**：ダッシュボード（概要） → キャンペーン一覧 → 詳細分析
- **デモシナリオ例**：「佐藤さんが月次レビュー前に、3つのキャンペーンの成果を比較し、最も効果的な施策を特定する」

## 🛠️ IBM Bob 使い方ガイドライン

### 初心者（LOBメンバー）向けのポイント

#### 1. 最初の一歩
```
「シンプルな顧客リスト画面を作って。
名前、年齢、電話番号を表示するテーブルを含めて。」
```

#### 2. 段階的に機能追加
```
「顧客リストに検索ボックスを追加して。
名前で絞り込みできるようにして。」
```

#### 3. デザインの改善
```
「もっと見やすくして。
ヘッダーを青色にして、テーブルに縞模様をつけて。」
```

#### 4. インタラクティブ機能
```
「顧客名をクリックすると、詳細情報が表示されるようにして。」
```

### 効果的なプロンプトのコツ

#### ✅ 良い例
- 「営業担当者向けの顧客リスト画面を作って。名前、年齢、家族構成を表示するカード・レイアウトで。」
- 「検索ボックスを追加して、名前で絞り込みできるようにして。」
- 「ヘッダーに会社ロゴとナビゲーション・メニューを追加して。」

#### ❌ 避けるべき例
- 「いい感じの画面を作って」（具体性がない）
- 「データベースと連携して...」（複雑すぎる）
- 「完璧なシステムを作って」（範囲が広すぎる）

### トラブルシューティング

#### よくある問題と対処法

1. **思った通りに動かない**
   - 具体的に何が違うかを説明して修正依頼
   - 「検索ボックスが動かない。名前を入力したら、リストが絞り込まれるようにして。」

2. **デザインが気に入らない**
   - 具体的な改善点を指示
   - 「ボタンをもっと大きく、青色にして。」

3. **機能を追加したい**
   - 一つずつ段階的に追加
   - まず基本機能を完成させてから、追加機能へ
