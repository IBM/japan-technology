## 📌 基本の使い方

### Bobとの対話の基本
1. **具体的に指示する**：「何を」「どのように」作りたいかを明確に
2. **段階的に進める**：一度に全部ではなく、少しずつ機能を追加
3. **確認しながら進む**：各ステップで結果を確認してから次へ

## 🎯 よく使うプロンプト集

### 1. 最初の画面を作る

#### シンプルなリスト画面
```
顧客リスト画面を作って。
以下の情報を表示するテーブルを含めて：
- 顧客名
- 年齢
- 電話番号
- 住所

サンプルデータを5件含めて。
```

#### カードレイアウト
```
顧客情報をカード形式で表示する画面を作って。
各カードには以下を含めて：
- 顧客名（大きく表示）
- 年齢と家族構成
- 最終訪問日
- 「詳細を見る」ボタン

3列のグリッドレイアウトで、サンプルデータを6件含めて。
```

#### ダッシュボード
```
マーケティングダッシュボードを作って。
以下の要素を含めて：
- ページタイトル「キャンペーン管理」
- 3つの統計カード（総リード数、コンバージョン率、進行中のキャンペーン数）
- 簡単な棒グラフ（月別リード数）
- キャンペーン一覧テーブル
```

---

### 2. デザインを改善する

#### 色とスタイル
```
以下のデザイン変更をして：
- ヘッダーの背景色を濃い青（#1e3a8a）に
- ボタンを緑色（#10b981）で角丸に
- テーブルに縞模様を追加
- 全体的に余白を広げて見やすく
```

#### レスポンシブ・デザイン
```
この画面をスマホでも見やすくして。
- カードを1列に並べる
- フォントサイズを調整
- ボタンを大きく、タップしやすく
```

#### プロフェッショナルな見た目
```
もっとプロフェッショナルな見た目にして：
- 洗練されたフォント（sans-serif）
- 適切な影（shadow）を追加
- アイコンを追加（Font Awesomeなど）
- ホバー効果を追加
```

### 3. インタラクティブ機能を追加

#### 検索・フィルター
```
検索機能を追加して：
- ページ上部に検索ボックス
- 顧客名で絞り込み
- リアルタイムで結果を更新
```

#### 詳細表示
```
顧客名をクリックすると、詳細情報がモーダルウィンドウで表示されるようにして。
詳細情報には以下を含めて：
- 基本情報（名前、年齢、連絡先）
- 家族構成
- 契約中の保険
- 訪問履歴（日付とメモ）
```

#### フォーム
```
住所変更フォームを作って：
- 郵便番号（7桁）
- 都道府県（ドロップダウン）
- 市区町村
- 番地
- 建物名（任意）
- 「確認」ボタンと「キャンセル」ボタン

入力後、確認メッセージを表示。
```

---

### 4. データ表示を工夫する

#### グラフ・チャート
```
月別のリード数を棒グラフで表示して。
- X軸：1月〜12月
- Y軸：リード数
- 各月のデータをサンプルで含めて
- グラフの上に合計値を表示
```

#### 進捗バー
```
オンボーディングの進捗を表示して：
- 全体の進捗率（パーセンテージ）
- 進捗バー（色付き）
- 完了したタスク数 / 全タスク数
- 「あと○個のタスクが残っています」というメッセージ
```

#### ステータス・バッジ
```
キャンペーンのステータスをバッジで表示して：
- 「進行中」→ 緑色
- 「準備中」→ 黄色
- 「終了」→ グレー
- 各バッジは角丸で、適切なパディング
```

```
ページ上部にナビゲーションヘッダーを追加して：
```

```
- タブ1：基本情報
```

#### パンくずリスト

```
パンくずリストを追加して：
ホーム > 顧客管理 > 顧客詳細
各項目はクリック可能に。
```

## 🎨 デザインのヒント

### 色の選び方
- **プライマリーカラー**：青系（信頼感）
  - 例：`#1e40af`, `#3b82f6`, `#60a5fa`
- **アクセントカラー**：緑系（アクション）
  - 例：`#10b981`, `#34d399`
- **警告色**：黄色・オレンジ系
  - 例：`#f59e0b`, `#fbbf24`
- **エラー色**：赤系
  - 例：`#ef4444`, `#f87171`

### レイアウトのコツ
```
以下のレイアウトにして：
- 最大幅：1200px
- 中央揃え
- 左右に適切な余白（padding: 20px）
- セクション間の余白（margin: 40px）
```

### フォントサイズ
```
フォントサイズを調整して：
- ページタイトル：32px、太字
- セクション見出し：24px、太字
- 本文：16px
- 補足情報：14px、グレー
```

## 🔧 よくある問題と解決法

### 問題1：思った通りに動かない
**解決法**
```
検索機能が動かない。
検索ボックスに名前を入力したら、
リストがリアルタイムで絞り込まれるようにして。
大文字小文字は区別しないで。
```

### 問題2：デザインが崩れる
**解決法**
```
レイアウトが崩れている。
以下を修正して：
- カードの高さを揃える
- 画像のサイズを統一（200x200px）
- グリッドの間隔を均等に（gap: 20px）
```

### 問題3：データが表示されない
**解決法**
```
顧客データが表示されない。
以下のサンプルデータを使って表示して：

1. 山田太郎、45歳、東京都、03-1234-5678
2. 佐藤花子、38歳、大阪府、06-9876-5432
3. 鈴木一郎、52歳、愛知県、052-111-2222
```

### 問題4：ボタンが反応しない
**解決法**
```
「詳細を見る」ボタンをクリックしても何も起こらない。
クリックしたら、その顧客の詳細情報を
モーダルウィンドウで表示するようにして。
```

## 🎯 業務領域別のプロンプト例

### 営業チーム
```
営業担当者向けの顧客訪問支援画面を作って。
以下の機能を含めて：

1. 今日の訪問予定リスト
   - 顧客名、訪問時間、住所

2. 顧客情報カード
   - 基本情報（年齢、家族構成、職業）
   - 前回の訪問メモ
   - おすすめ商品

   - 「訪問完了」ボタン
   - 「メモを追加」ボタン
   - 「次回予定を設定」ボタン

モバイルファーストで、見やすく使いやすいデザインに。
```

```
キャンペーン管理ダッシュボードを作って。

1. KPIサマリー（4つのカード）

   - 名前、期間、ステータス、進捗率
```

```
契約者向けのマイページを作って。

3. クイックアクション
   - 住所変更
   - 保険金請求

4. お知らせ
   - 重要なお知らせを3件表示

安心感のある、落ち着いたデザインに。
```

```
新入社員向けのオンボーディング画面を作って。
```

## 💡 プロのようなUXを作るコツ

### 1. ローディング表示
```
データ読み込み中は、ローディングスピナーを表示して。
「データを読み込んでいます...」というメッセージも追加。
```

### 2. 空の状態
```
データがない場合は、以下を表示して：
- アイコン（空のボックスなど）
- 「まだデータがありません」
- 「新規作成」ボタン
```

### 3. 成功メッセージ
```
フォーム送信後、画面上部に成功メッセージを表示して：
- 緑色の背景
- チェックマークアイコン
- 「保存しました」というテキスト
- 3秒後に自動で消える
```

### 4. 確認ダイアログ
```
削除ボタンをクリックしたら、確認ダイアログを表示して：
- 「本当に削除しますか？」
- 「この操作は取り消せません」
- 「削除」ボタン（赤）と「キャンセル」ボタン（グレー）
```
