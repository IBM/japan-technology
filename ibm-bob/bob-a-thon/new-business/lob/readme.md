# Bob-a-thon (AIワークショップ)
## ビジネス担当者向け - 新規事業立ち上げシナリオ
* 最終更新日: 2026/4/6
* Author: Akira Onishi (Principal BDE, AI Lab Japan Office, IBM Japan)

## 📋 ワークショップ概要

### 目的
- **AI駆動開発のパワーを体感**：IBM Bobを使った開発の速さと可能性を実感
- **ユーザー体験重視の開発**：コードの正確性よりもUXの素晴らしさを追求
- **ビジネス側とIT側の視点の確認**：異なる視点からのアプローチを比較し、相互理解を深める
- **実践的なプロトタイピング**：短時間で動くモックアップを作成

### 対象業界
このワークショップは業界を問わず適用可能です。以下のような業界での活用を想定しています：

- **SaaS/IT**: プロジェクト管理ツール、CRM、マーケティングツール
- **EC/小売**: オンラインショップ、D2Cブランド、オーガニック食品
- **製造業**: IoTセンサー、産業機械、部品メーカー
- **サービス業**: オンライン教育、コンサル、人材サービス
- **金融**: フィンテック、決済サービス、保険

参加者の業界・業種に合わせて、企業設定や商品/サービスをカスタマイズしてください。

### 状況の想定
* 少人数で新規事業「[企業名]（仮）」を立ち上げます。一人ひとりがリーダーであり、実務家です。
* テクノロジーを活用した新規事業という設定です。AI駆動開発を採用し、IBM Bobが皆さんのパートナーです。
* IBM Bobの使い方については、[IBM Bob 使い方ガイドライン](#ibm-bob)を参考にしてください。
* ワークショップ開始時に、参加者で相談してIBM Bobと壁打ちしながら以下を決定してください：
  - 業界・業種
  - 企業名
  - 主要商品/サービスの名称
  - ミッション、ビジョン、バリュー
  - ブランドカラー・フォント
* 参加者同士で企業コンセプトを合意・共有しておくと、ワークショップの作業がより実践的になります。

**設定例:**
- SaaS企業（プロジェクト管理ツール「TaskFlow」）
- EC事業（オーガニック食品通販「GreenMarket」）
- 教育サービス（オンライン研修プラットフォーム「SkillUp」）
- 製造業（IoTセンサー機器「SmartSense」）

* IBM Bobの操作がわからない、モックアップの修正において困ったことがあれば、ワークショップのサポーターにお声がけください。

### The Geek Way (ギーク思考）について
* 参考書籍: [ギーク思考 圧倒的な結果を出す型破りな思考法](https://bookplus.nikkei.com/atcl/catalog/25/03/25/01931/)
- ギーク思考の４つの規範について、ワークショップを通じて体感してください。
* スピード: 綿密な計画よりも、結果を出すための素早い反復

  まずは試してみて、上手くいかなかったら、別のアプローチを試してみるようにします。上手くいったら、さらに改善を試みます。
  特にIBM Bob (AIエージェント)との会話を何度も往復することが重要です。
  
* オーナーシップ: 共通のゴールを目指して、自律的に活動

  少人数で新規事業を立ち上げる設定です。一人ひとりがリーダーであり、実務家です。
  業務のイメージを固めるためにモックアップを作りましょう。IBM Bobが皆さんのパートナーです。
 
* サイエンス: 実験からデータを集め、証拠を元に議論を実践

  短時間のワークショップなので、詳細なデータを集めるというよりは、試行した結果から、よりブラッシュアップすることを意識してください。

* オープンネス: 情報を徹底して共有し、議論や方向転換を歓迎
 
 アイディアを共有する、出来上がったモックアップを他の人に操作してもらう、など、周りからフィードバックをもらうことを心がけてください。
 またフィードバックを求められたら、積極的に参加して、より良くするための意見を伝えましょう。


## 🎯 各業務領域の課題設計

### 設計方針
1. **短時間で完成可能な範囲**：シンプルだが価値のある機能に絞る
2. **ユーザー体験重視**：見た目の美しさ、使いやすさを評価
   
   複雑になるのを避けるため、「ブラウザーでHTMLファイルを開くだけで確認できるモックアップとして作成して」とBobに伝えながらワークショップを進めてください。
   ITスキルが活用できる方は、Webアプリとして作成しても構いません。

3. **実務との関連性**：実際の業務課題を反映

   課題カードには最低限の内容を書いてあります。皆さんの業務課題を踏まえながら、適宜、改良を加えてください。
  
4. **創造性の余地**：正解が一つではない、工夫の余地がある課題

### 1. 営業チーム：「顧客関係管理（CRM）システム」

課題カードの内容を実現してください。最適な商品/サービスの部分については、新たな商品を定義しながら、発展させてください。

#### 課題カード
```
【営業チームの課題】

新規事業として、営業担当者が効率的に顧客対応を行い、
適切な商品/サービスを提案できるシステムが必要です。

■ ペルソナ
- 営業担当者：田中さん（35歳、営業経験5年）
- 1日に3-5件の顧客対応をこなす
- 移動中や商談前に顧客情報を確認したい

■ 解決したい課題
- 顧客の基本情報を素早く確認
- 顧客に最適な商品/サービスを提案
- 対応履歴と次回アクションを記録

■ 必須機能（最低限これだけは実現）
1. 顧客リスト表示（名前、属性、ステータス）
2. 顧客詳細画面（基本情報 + 対応履歴）
3. おすすめ商品/サービスの表示

■ あると嬉しい機能（時間があれば）
- 顧客の検索・フィルター機能
- 対応予定のカレンダー表示
- 簡単な提案資料生成

■ UX重視ポイント
- モバイルでも見やすいデザイン
- 情報が一目でわかるビジュアル
- 直感的な操作性

■ 業界別適用例
- SaaS: 顧客企業の利用状況、アップセル提案
- 製造: 取引先の発注履歴、在庫状況
- 小売: 顧客の購買履歴、おすすめ商品
- コンサル: クライアントのプロジェクト状況、提案内容
```

#### 期待される成果物
- **画面構成**：顧客リスト → 顧客詳細 → 商品/サービス提案
- **デモシナリオ例**：「田中さんが午後の商談前に、山田様（45歳、家族4人）の情報を確認し、ニーズに合った商品/サービスを提案する」

**業界別シナリオ例:**
- SaaS: 利用状況を確認し、上位プランへのアップグレードを提案
- 製造: 発注履歴を確認し、新製品ラインナップを紹介
- 小売: 購買傾向を分析し、季節商品をレコメンド
- コンサル: プロジェクト状況を確認し、追加サービスを提案

#### オプション課題（時間に余裕がある場合)

営業活動の全体像を把握するダッシュボードを作成してください。
（商談状況、提案数、成約率、パイプライン管理など）

### IBM-Bob
## 🛠️ IBM Bob 使い方ガイドライン

### 初めてIBM Bobを利用する人向けのポイント

#### 1. 最初の一歩
```
「シンプルな顧客リスト画面を作って。
名前、年齢、電話番号を表示するテーブルを含めて。」
```

#### 2. 段階的に機能追加
```
「顧客リストに検索ボックスを追加して。
名前で絞り込みできるようにして。」
```

#### 3. デザインの改善
```
「もっと見やすくして。
ヘッダーを青色にして、テーブルに縞模様をつけて。」
```

#### 4. インタラクティブ機能
```
「顧客名をクリックすると、詳細情報が表示されるようにして。」
```

### 効果的なプロンプトのコツ

#### ✅ 良い例
- 「営業担当者向けの顧客リスト画面を作って。名前、属性、ステータスを表示するカード・レイアウトで。」
- 「検索ボックスを追加して、名前で絞り込みできるようにして。」
- 「ヘッダーに会社ロゴとナビゲーション・メニューを追加して。」

#### ❌ 避けるべき例
- 「いい感じの画面を作って」（具体性がない）
- 「データベースと連携して...」（複雑すぎる）
- 「完璧なシステムを作って」（範囲が広すぎる）

### トラブルシューティング

#### よくある問題と対処法

1. **思った通りに動かない**
   - 具体的に何が違うかを説明して修正依頼
   - 「検索ボックスが動かない。名前を入力したら、リストが絞り込まれるようにして。」

2. **デザインが気に入らない**
   - 具体的な改善点を指示
   - 「ボタンをもっと大きく、青色にして。」

3. **機能を追加したい**
   - 一つずつ段階的に追加
   - まず基本機能を完成させてから、追加機能へ
