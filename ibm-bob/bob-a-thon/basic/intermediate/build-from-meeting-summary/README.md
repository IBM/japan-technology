# Bob-a-thon Project 5 : ビジネスユーザー向けの仕様開発


## ハンズオンの目的

本ハンズオンでは、IBM Bobが単なるコード生成ツールではなく、「ビジネスユーザーとソフトウェア開発チームの架け橋」として機能し、エンタープライズ規模のユースケース実装を加速させるプロセスを体感していただきます。

  
## 背景設定
税務申告（W-2フォーム）のミスにより、申告のやり直しが発生したり、税務当局から指摘を受けたりしているという課題があります。
業務部門において、この課題を整理して根本的な改善策を探るための会議が行われました。その議事録が [会議メモ.md](/会議メモ.md) ファイルです。この議事録をもとに、税務申告の前にデータがエラーを含んでいないかを検出するアプリケーションのプロトタイプ開発を行います。
  
  
## ハンズオン手順

IBM Bobに適切なプロンプトを与えながら各ステップを進めていきます。どのようなプロンプトを与えればいいのか考えてみましょう。（プロンプトの例は[こちら](#プロンプト例)

### Step 1: 業務要件の整理
議事録の内容を踏まえて、ビジネス要件をまとめたドキュメントを作成します。
Bobに会議メモを渡して、その要点を整理し、目的、対象ユーザー、必要な機能要件などをまとめた ビジネス要件定義書 を生成させてください。

期待される結果 : 
生成された要件定義書を確認してください。会議メモで挙げられていた問題が要件として盛り込まれていることを確認します。

  
### Step 2: モックアップでUIの確認
ビジネスユーザーが 簡単な画面モックアップを作ることにより、ユーザー体験を早期に確認できます。
Bobに指示して税務申告書の検証システム用のシンプルなモックアップを HTMLとJavaScript で生成させてください。

期待される結果 : 
Webブラウザを起動して生成されたファイルを開いてください。入力フォームや検証ボタンなどを備えたモックアップが生成されていることを確認します。

  
### Step 3: アーキテクチャ仕様書の作成
作成されたビジネス要件を満たすための技術的な設計図（アーキテクチャ）を作成します。
Bobにシニアエンジニアとしての役割を与え、専門的なアーキテクチャ仕様書を生成させてください。

期待される結果 : 
グラフィカルなアーキテクチャ図を含む仕様書が生成されていることを確認してください。

  
### Step 4: 実装仕様書の作成
アーキテクチャが固まったら、プログラマーが実際にコードを書くための詳細な「実装仕様書」へと落とし込みます。
Bobにこれまでの出力をもとに詳細な実装仕様書を生成させてください。

期待される結果 : 
API仕様やクラス設計を含む技術仕様書が生成されていることを確認してください。

  
### Step 5: プロトタイプの生成とテスト
実装仕様書にもとづいて、Bobに初期のプロトタイプコードを生成させてください。

期待される結果 : 
アプリケーションを実行し、仕様通りに動作するかを確認してください。不具合がある場合には追加で指示を行って適宜修正してください。

  
### Step 6: SDLCのプロセスを進める
SDLC（ソフトウェア開発ライフサイクル）の残りの工程を進めます。

期待される結果 : 
セキュリティレビュー、デプロイスクリプトの作成など、開発ライフサイクルのあらゆる場面で活用できることをご確認ください。

## まとめ

IBM Bobを使って、ビジネス部門の「要望（自然言語）」を、IT部門の「仕様（技術言語）」に瞬時に翻訳するプロセスを体験いただきました。
Bobを活用することで部門間のコミュニケーションのズレを防いで手戻りをなくし、コストと時間の削減につながることをご理解いただけましたでしょうか。
