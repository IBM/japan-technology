# クレジットカード管理マルチエージェントアプリケーション プロトタイプ

## 概要
Watsonx Orchestrateで動作するマルチエージェントクレジットカード管理アプリケーションのプロトタイプを作成してください。

これはプロトタイプであるため、アプリケーションのいくつかのコンポーネントをハードコードします。サンプルのクレジットカード取引とツールの戻り値をハードコードします。
サンプル取引は、小売、飲食、食料品、サブスクリプション、旅行、ガソリン、公共料金など、さまざまなカテゴリから選択してください。

## エージェント実装の詳細

### 1. エージェント設定フォーマット
エージェントの指示はYAMLで実装されます。このWebページのYAMLガイドラインを使用してください: https://developer.watson-orchestrate.ibm.com/agents/build_agent

### 2. エージェントアーキテクチャ
3つのエージェントを作成します:
- `fraud_agent` - 不正検知とカードセキュリティを処理
- `manage_subscriptions_agent` - サブスクリプション関連の取引を管理
- `credit_card_management_agent` - 他の2つのエージェントを調整するオーケストレーターエージェント

### 3. エージェント仕様

#### 3.1 fraud_agent
**責任:** 異常な取引に関する質問への回答、クレジットカードの凍結またはキャンセル、新しいクレジットカードのリクエスト。

**ツール:**
- `check_transactions()`: 異常な取引をリストします。不正の可能性がある取引を最大3件返す必要があります。
- `report_fraud(transaction_id: str)`: 不正ケースを開き、新しいクレジットカードの送付を推奨します。入力として取引IDが必要です。
- `freeze_credit_card(card_number: str)`: クレジットカードが凍結されたことの確認を返します。入力としてカード番号(下4桁)が必要です。
- `cancel_credit_card(card_number: str, reason: str)`: クレジットカードがキャンセルされたことの確認を返します。カード番号と理由が必要です。
- `send_new_credit_card(address: str)`: 新しいクレジットカードが送付されたことの確認を返します。配送先住所が必要です。

#### 3.2 manage_subscriptions_agent
**責任:** サブスクリプションに関連するすべての請求のリスト表示とサブスクリプションキャンセルの管理。

**ツール:**
- `get_subscriptions()`: サブスクリプションであるすべての取引をリストします。詳細(名前、金額、頻度、次回請求日)を含む5〜7件のサブスクリプションを返す必要があります。
- `get_subscription_spend(category: str = None)`: さまざまなサブスクリプションカテゴリに費やした金額を計算します。オプションのカテゴリフィルター(例: "streaming"、"software"、"fitness")。
- `cancel_subscription(subscription_name: str)`: サブスクリプションがキャンセルされたことの確認を返します。サブスクリプション名が必要です。

#### 3.3 credit_card_management_agent
**責任:** fraud_agentとmanage_subscriptions_agentを調整して、統一されたクレジットカード管理エクスペリエンスを提供します。

**機能:**
- ユーザークエリを適切な専門エージェントにルーティング
- 必要に応じて複数のエージェントからの情報を組み合わせ
- クレジットカード管理のための会話型インターフェースを提供


### 4. ツール実装ガイドライン

#### 4.1 プログラミング言語
ツールはWatsonx Orchestrate SDKを使用してPythonで実装されます。

#### 4.2 ツール構文テンプレート
```python
# example_tool.py
from ibm_watsonx_orchestrate.agent_builder.tools import tool


@tool()
def my_tool(input: str) -> str:
    """提供された入力に基づいてツールのアクションを実行します。

    Args:
        input (str): ツールの入力。

    Returns:
        str: ツールのアクション。
    """
    # ツールの機能
    return f"Hello, {input}"
```

#### 4.3 ハードコードデータ要件
- すべてのツールは、このプロトタイプのためにハードコードされた値を返す必要があります
- サンプルデータは現実的で多様である必要があります
- 戻り値に適切なデータ構造(辞書、リスト)を含める
- タイムスタンプ、取引ID、その他の関連メタデータを追加

### 6. 成果物

#### 6.1 テストWeb UI
すべてのツールを個別にテストできるテストWeb UI(HTML/JavaScript)を生成します:
- 適切なパラメータを持つ各ツールの入力フォーム
- 読みやすい形式でツールの応答を表示
- デモンストレーション目的のシンプルでクリーンなインターフェース

#### 6.2 ドキュメント(README.md)
マークダウンファイルには以下を含める必要があります:

**a. 実装の詳細**
- アーキテクチャの概要
- エージェントの説明と責任
- 入出力仕様を含むツールの説明
- サンプルのハードコードデータ構造

**b. デプロイメント手順**
- 前提条件(Pythonバージョン、SDKインストール)
- ツールインポートコマンド: `orchestrate tools import -k python -f <file_name>.py`
- エージェントインポートコマンド: `orchestrate agents import -f <file_name>.yaml`
- ステップバイステップのデプロイメントガイド

**c. サンプル質問(5〜10例)**
例には以下を含める必要があります:
- 不正検知クエリ
- サブスクリプション管理クエリ
- オーケストレーションを必要とする複合クエリ
- エッジケースとエラーシナリオ

**サンプル質問:**
1. "カードに異常な取引があれば表示してください"
2. "どのサブスクリプションに支払っていますか?"
3. "ストリーミングサービスにいくら使っていますか?"
4. "XYZストアからの不審な請求があります。助けてもらえますか?"
5. "ジムのメンバーシップサブスクリプションをキャンセルしてください"
6. "すぐにクレジットカードを凍結してください"
7. "月間のサブスクリプション総額はいくらですか?"
8. "不正取引を報告して新しいカードを送ってください"
9. "すべてのエンターテインメントサブスクリプションを表示してください"
10. "カードを紛失しました。キャンセルして代替品を送ってください"

### 7. ベストプラクティス

- 明確で説明的な関数名と変数名を使用
- すべてのツールに包括的なdocstringを含める
- すべての関数パラメータと戻り値に型ヒントを追加
- PythonコードにはPEP 8スタイルガイドラインに従う
- YAMLファイルで一貫したフォーマットを使用
- ツール実装にエラーハンドリングを含める(ハードコードされた値を返す場合でも)
- ハードコードされたデータ構造を説明するコメントを追加
