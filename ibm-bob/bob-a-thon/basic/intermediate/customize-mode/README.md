# Project 8: watsonx.ai developer mode を作成してみよう！

## 開発者向けハンズオン
- **誰向けか**: IBM Bob / カスタムモードを使って、watsonx.ai API 向けのコード生成・改善を行いたい開発者
- **想定所要時間**: 20分〜30分
- **最終更新日**: 2026/4/8

## 📋 概要

このハンズオンでは、IBM Bob の **Custom Mode** 機能を使って、**watsonx.ai developer mode** を作成し、実際に watsonx.ai API を利用するコード生成を試します。

既存モードの確認、モード案のブレインストーミング、モード定義の作成・調整、そして生成したモードのテストまでを一通り体験することで、特定用途に最適化された AI 支援開発の進め方を学びます。

## 🎯 目的

- IBM Bob の **Custom Mode** の基本的な考え方と使い方を理解する
- watsonx.ai API に特化したモード定義を作成できるようになる
- コード生成前に必要な確認事項（言語、SDK/REST、モデル、応答形式など）を整理できるようになる
- watsonx.ai を利用するアプリケーション開発の初期セットアップを理解する
- 実務で再利用できるカスタムモードの設計・改善方法を習得する

## 💼 状況の想定

あなたは、生成 AI を活用したアプリケーション開発を進めている開発者です。
チームでは watsonx.ai API を活用したサンプルコードや検証コードを効率良く作成したいと考えており、IBM Bob の Custom Mode を使って、watsonx.ai 向けの問い合わせ・コード生成を標準化することになりました。

まずは、watsonx.ai API に特化した **developer mode** を作成し、実際に Python もしくは Node.js のコード生成を行える状態を目指します。

## 📦 本ハンズオンを開始する前に

このハンズオンをスムーズに進めるために、以下の準備を行ってください。

### 必須事項
- **watsonx.ai のアカウントを事前に作成していること**
- **IBM Bob が利用可能な状態であること**
- **IBM Bob にアクセスできるアカウントを持っていること**
- **watsonx.ai の API Key を取得済みであること**
- **watsonx.ai の Project ID を取得済みであること**

> **重要**  
> 本ハンズオンでは、watsonx.ai を利用するコード生成やテストを行うため、**API Key** と **watsonx project id** が必須です。事前に取得しておいてください。

### 事前に確認しておくとよい情報
- 使用したい言語（**Python SDK** または **Node.js**）
- 実装方式（**REST 呼び出し** または **SDK API**）
- 利用したい **LLM / モデル ID**
- 応答形式（**テキスト** または **ストリーミング**）
- watsonx.ai の接続先 URL

### 推奨環境
- Web 検索が利用可能な IBM Bob 環境
- `.env` ファイルを作成できるローカル開発環境
- Python または Node.js の実行環境

### 確認事項
- [ ] IBM Bob が起動できる
- [ ] **Settings > Modes** を開ける
- [ ] Custom Mode を新規作成できる
- [ ] watsonx.ai の **API Key** を確認できる
- [ ] watsonx.ai の **Project ID** を確認できる
- [ ] 必要に応じて `.env` ファイルを作成できる

## 🧭 ハンズオンの進め方（概要）

1. **既存モードを確認する**  
   Settings > Modes から、標準で用意されているモードとその説明を確認します。

2. **Bob とモード案をブレインストーミングする**  
   watsonx.ai API 向けに、どのような確認項目や出力ルールが必要かを整理します。

3. **watsonx.ai developer mode を作成する**  
   ドラフト定義をベースに、Bob にレビュー・改善を依頼しながらモード定義を仕上げます。

4. **モードをテストする**  
   生成したモードを使って、watsonx.ai API を試すためのスクリプト生成を行います。

## ✨ このハンズオンで学べること

- 汎用モードではなく、**用途特化型モード** を作るメリット
- watsonx.ai API 用コード生成時に必要な入力項目の整理方法
- `.env` を使った設定値の安全な切り分け方
- チーム内で再利用しやすいモード定義の考え方
