# Bob デモプロジェクト - SDLC ショーケース

## 概要
このプロジェクトは、Bob IDE のさまざまな機能や能力を紹介するために作られています。現実的な銀行アプリケーションのシナリオを通じて、コード理解、セキュリティ分析、コンプライアンスレビュー、インフラ自動化、コード生成における Bob の強みを実演できます。

## デモシナリオ

### 1. アプリケーション理解 (`1_app_understanding/`)
**目的:** ドキュメント化されていないコードを分析し、説明する Bob の能力を実演する。

**シナリオ:** 最小限のドキュメントしかない 2 つの銀行アプリケーションが含まれています。
- **bank1-savings** - モダンな React フロントエンドを備えた主要な銀行です。ベンダーがこのアプリケーションを構築しましたが、何も文書化しないまま離任しました。まず Bob に、このアプリケーションの仕組み、アーキテクチャ、主要機能を説明させてください。
- **bank2-investment** - 古い UI を持つ旧式の銀行です。UI モダナイゼーションのシナリオ（例: bank1 が bank2 を買収し、インターフェースを更新する必要がある場合）や、銀行システム間のコンテナ間通信の実演に使用できます。

### 2. セキュリティ & コンプライアンスレビュー (`2_security_review/`)
**目的:** Bob のセキュリティ分析およびコンプライアンス対応能力を紹介する。

**シナリオ:** `bank1-savings` には複数のセキュリティおよびコンプライアンス上の問題が含まれています。
- `app_vulnerable.py` を開き、余白に表示されるバグアイコンから、Bob が検出したセキュリティ不具合を確認してください
- Bob のリファクタリング提案を示すヘルメットアイコンを確認してください
- チャットで Bob に詳細なセキュリティ調査を依頼してください
- 段階的な是正計画（必須修正と GA 後の更新）を依頼してください
- `app_sox_issues.py` を使って SOX コンプライアンス分析を実演してください

### 3. インフラ自動化 (`3_banking_demo_automation/`)
**目的:** Infrastructure as Code（IaC）を生成し、活用する Bob の能力を実演する。

**シナリオ:** これを Ansible および Terraform 自動化の実演の基盤として使用してください。
- ローカル環境へのデプロイ先（Docker、Rancher Desktop）
- クラウド向けデプロイ設定（Azure）
- 理想的には、より初期のスナップショットから始めて、Bob にこれらの自動化をその場で生成させてください
- あるいは、Bob が自動化作業を完了した後に、このディレクトリを参照用として使用してください
