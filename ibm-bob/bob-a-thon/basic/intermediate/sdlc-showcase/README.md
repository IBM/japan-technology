# Project 2: SDLCタスク
- 対象者: 開発者向け

## 概要
このハンズオンでは、Bob を使って SDLC のさまざまなタスクを実施します。ドキュメントが十分ではない既存アプリケーションを題材に、理解、修正、レビュー、拡張、デプロイ確認までを段階的に体験します。題材となるのは、口座概要の確認、取引履歴の参照、銀行間振込、支出や投資の分析などを行うバンキングアプリケーションです。

主な内容には、既存コードベースのレビューとドキュメント化、新機能の追加、更新したアプリケーションのデプロイ確認、セキュリティレビューの実施、バグ修正、が含まれます。

## 目的
- Bob を使って既存コードベースを理解し、必要な情報を整理する
- アプリケーションの起動やデプロイ確認まで実施する
- `spec development` を使って新機能を追加する
- セキュリティやコンプライアンスの観点でレビュー、バグ修正を行う

- 想定所要時間: 1〜2時間
- 想定コスト: $2〜$3

## 状況の想定
あなたは、十分なドキュメントが残っていない銀行アプリケーション群を引き継いだ開発者です。Bob を活用しながら、アプリケーションの理解、課題の修正、セキュリティレビュー、機能追加、デプロイ確認までを進めます。実務に近い流れの中で、Bob を使った開発支援の進め方を体験することが目的です。

## 本ハンズオンを開始する前に
- 必須
  - IBM Bob が利用可能な状態であること
  - ターミナルとブラウザが利用できること
- 実施内容に応じて必要
  - Python 実行環境
  - Node.js 実行環境
- 任意
  - Docker
  - Podman
  - Rancher Desktop などのコンテナ環境
- 補足
  - 詳細なセットアップや操作手順は、各シナリオ配下の手順に従ってください

## 手順
### `1_app_understanding`

1. この Git repo を、Bob またはお使いの Git ツールでダウンロードします。
2. `Project2_SDLC_tasks\1_app_understanding` フォルダを開きます。
3. Bob を使ってアプリケーションを説明させます。
   - Advancedモードにセットしてください。
   - 汎用的なプロンプトでも、より具体的なプロンプトでも構いません。
   - プロンプト例: `このアプリケーションを説明してください。アプリケーション図、使用されている API、前提条件、セットアップ手順、起動手順を含めてください。`
4. 説明内容を `.md` ファイルとして保存します。
   - プロンプト例: `この説明を Markdown で保存してください。`
   - その後も Bob に追加の質問をします。たとえば、使用されているライブラリについて質問します。
5. セットアップ手順と起動手順を確認します。
   - 手順が詳しすぎる場合は、インストール/前提条件と起動手順だけが分かるように、追加で質問して明確化します。
   - サーバー起動時はポートに注意してください。正しいポートは Python のターミナル出力に表示されます。
   - エラーが発生した場合は、Bob を使ってトラブルシュートします。
6. Bob に、アプリケーションを起動するスクリプトの作成を依頼します。
   - プロンプト例: `このアプリケーションをセットアップして起動してください。`
   - 承認を求められた場合は、「実行」や「実行中も続行」ボタンを押して進めてください。
   - 完了後にブラウザを開いてアプリが起動されたことを確認します。
7. `spec development` を使って、アプリケーションに新機能を追加します。
   - たとえば、`Promotions` タブを追加します。
   - プロンプト例：
      - spec developmentで`Promotions` タブを追加します。
      - Promotions はデータベースから読み込むようにします（他のアプリケーション部分と同様に SQLite を使用できます）。
      - Promotions は、口座残高やその他のデータポイントに関連する内容にします。
8. 【オプション Docker, Podman、Rancher、またはノート PC 上で使っているその他のコンテナプラットフォームがある場合】Bob に、ローカルのコンテナ環境向けのコンテナデプロイを作成するよう依頼します。
   - プロンプト例: `ローカルのコンテナ環境向けのコンテナデプロイを作成してください。`
   - デプロイをテストします。


### `2_security_review`

このセクションでは、セキュリティ上の問題を含むアプリケーションを扱います。

1. `Project2_SDLC_tasks\2_security_review` フォルダを開きます。
2. Bob を使って、セキュリティ問題を特定する能力を確認します。アプリケーションセキュリティの専門家であれば、自分のプロンプト群を使っても構いません。
   - サンプルプロンプトは `Project2_SDLC_tasks\2_security_review\sample_prompts.txt` に保存されています。
3. Bob に `app_vulnerable.py` のセキュリティ問題を修正させます。
   - プロンプト例：`優先度の高い課題をまず修正してください。`
   - 変更内容をレビューします。
